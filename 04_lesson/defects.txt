1. Дефект: Некорректная обработка None в методах
Методы: contains(), delete_symbol()

Серьезность: Высокая (High)
Приоритет: Высокий (High)

Описание:
При передаче None в качестве аргумента в методы contains() и delete_symbol() возникает TypeError (из-за вызова string.index()), хотя ожидается явное выбрасывание AttributeError.

Шаги воспроизведения:

1)Вызвать contains(None, "a")

2)Вызвать contains("abc", None)

3)Вызвать delete_symbol(None, "a")

4)Вызвать delete_symbol("abc", None)

Фактический результат:
TypeError: index() argument 1 must be str, not None

Ожидаемый результат:
AttributeError: Arguments cannot be None

Статус: Исправлено
Исправление:
Добавлены явные проверки на None и выбрасывание AttributeError.

2. Дефект: Метод contains() работает с подстроками вместо символов
Серьезность: Средняя (Medium)
Приоритет: Средний (Medium)

Описание:
Метод contains() по документации должен проверять наличие одного символа, но фактически работает с подстроками (например, contains("SkyPro", "Sky") возвращает True).

Шаги воспроизведения:

1)Вызвать contains("SkyPro", "Sky")

Фактический результат:
True

Ожидаемый результат:
False (так как "Sky" — подстрока, а не символ)

Статус: Исправлено
Исправление:
Заменена реализация с string.index(symbol) на symbol in string.

Дефект: Неполная документация для capitalize()
Серьезность: Низкая (Low)
Приоритет: Низкий (Low)

Описание:
В документации capitalize() не описано поведение для:

Пустой строки (""),

Строк с числами ("123abc"),

Строк из одного символа ("s").

Шаги воспроизведения:

Вызвать capitalize("")

Вызвать capitalize("123abc")

Вызвать capitalize("s")

Фактический результат:
Нет четкого описания ожидаемого поведения.

Ожидаемый результат:
Документация должна явно описывать все граничные случаи.

Статус: Исправлено
Исправление:
Добавлены примеры в docstring.
